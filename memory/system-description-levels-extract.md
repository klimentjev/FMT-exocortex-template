# Экстракт: уровни описания системы

> Короткая версия в согласии с холонным подходом, ZP, FPF и SPF.

## Формула

`смысл -> модель -> холонная архитектура -> работа системы -> процедуры -> роли -> должности -> экземпляры -> данные`

## 9 уровней

| Уровень | Что описываем | Чего там не должно быть |
|---------|---------------|--------------------------|
| `0` | Смысл, целевое событие, базовый инвариант | Должностей, регламентов, кейсов |
| `1` | Сущности, различения, словарь, отношения | Конкретных исполнителей и расписаний |
| `2` | Холонную архитектуру: границы, контуры, интерфейсы, роль частей в целом | Микрошагов процессов и должностных инструкций |
| `3` | Операционные циклы, сигналы, артефакты, ритмы, точки управления | Штатного закрепления и персональных особенностей |
| `4` | Процессы, сценарии, протоколы, handoff'ы, условия переходов | Подмены архитектуры процессом |
| `5` | Роли, ответственности, права решений, владельцев контуров, эскалации | Смешения роли с должностью, агентом или инструментом |
| `6` | Должности, оргструктуру, закрепление ролей за людьми и командами | Онтологии и архитектурных инвариантов |
| `7` | Конкретный запуск, кейс, когорту, занятие, решение | Выведения общей модели из единичного случая |
| `8` | Метрики, события, журналы, индикаторы, подтверждающие материалы | Подмены данных интерпретацией |

## Быстрая привязка

- `Процесс` -> уровни `3-4`
- `Роль` -> уровни `2-5`
- `Должность` -> уровень `6`
- `Кейс / запуск` -> уровень `7`
- `Метрики / сигналы / логи` -> уровень `8`

## Жёсткие различения

- `Роль != Должность`
- `Архитектура != Процесс`
- `Операционный цикл != Конкретный запуск`
- `Данные != Модель`
- `Контур != Подразделение`
