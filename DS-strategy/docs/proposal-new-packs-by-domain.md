# Предложение: новые Pack'и по предметным областям

> Governance: структура Pack-репо. Знания в Pack'ах всеобщи (диссертация, статьи, доклады). Текущая проблема: в PACK-aufheben смешаны несколько предметных областей. Pack должен быть один домен = один Pack.

## Действия (по шагам)

1. **Путь:** доменные Pack'и только в **`PACK/PACK-<имя>/`** (логика, история философии, философия познания — уже в `PACK/`).
2. **Перенести** карточки из PACK-aufheben в соответствующие Pack'и с переименованием (префикс AUF → LOG / HFP / COG).
3. **Обновить** перекрёстные ссылки (related) в оставшихся карточках: вместо AUF.ENTITY.002 ссылаться на LOG.ENTITY.001 и т.д.
4. **Обновить** манифест PACK-aufheben: сузить границы до «процедура снятия и ЭП».
5. **Добавить** в navigation.md (и при необходимости в memory) новые Pack'и.

## Итоговая схема

```
PACK/PACK-aufheben     — процедура снятия, ЭП, этапы, типология, законы/критерии
PACK/PACK-logic        — формальная/содержательная логика, релевантная/паранепротиворечивая, противоречия
PACK/PACK-history-of-philosophy — три фазы ИФ, античные источники, момент/форма
PACK/PACK-philosophy-of-cognition — РП/РМ/РФП, субъект, условия помышления
PACK/PACK-education    — методика преподавания, освоение процедуры
PACK/PACK-bibliography — без изменений
PACK-personal (внешний/архив) — см. DS-strategy/docs/school/
```

После переноса в каждом Pack — один bounded context и одна предметная область.
