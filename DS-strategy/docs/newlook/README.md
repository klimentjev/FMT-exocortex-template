# Newlook Docs (Документы Newlook)

Эта папка содержит внутренние стратегические документы по продукту `Новый взгляд`.

## Каноника
`Internal-Product-Strategy-v2.md` - канонический документ и `source of truth` (канонический источник истины) по продукту.

Сначала меняется он, потом при необходимости синхронизируются производные документы.

## Производные документы
- `ICP.md` - ось `Customer` (клиент): портрет целевого клиента
- `Offer-Architecture.md` - оси `Trajectory` (траектория) + `Offer Logic` (логика оффера): продуктовая траектория и логика оффера
- `Program-Logic.md` - ось `Program Logic` (логика программы): как должна быть устроена программа, чтобы происходило изменение
- `Retention-Design.md` - ось `Retention Logic` (логика удержания): логика удержания и незаменимости клуба
- `Product-Hypotheses.md` - ось `Hypotheses To Prove` (гипотезы к доказательству): что именно нужно доказать до масштабирования

## Дополнительные материалы
- `Early-Concept-2026.md` - ранний концепт продукта и годовой рамки 2026; полезен как исторический входной материал, но не является текущим `source of truth`
- `Season-2026-Architecture.md` - короткая архитектурная записка по сезону 2026: повторяемый входной интенсив для новых, клуб как контур удержания и углубления, отдельные интенсивы только для участников клуба
- `Program-2026.md` - отдельная собранная программа места `Новый взгляд` на 2026 год в новой логике сезона: повторяемый квартальный вход, клубный контур, возможные специальные интенсивы для клубников
- `Revenue-Model-2026-2028.md` - отдельная расчетная модель дохода `Апр 2026 - Дек 2028`, выведенная из исходных параметров консервативного сценария
- `Economics-Model-2026-2028.md` - отдельная модель полной экономики `Апр 2026 - Дек 2028`: выручка, расходы, партнер, автор и накопление

## Следующий слой документов
Стратегический слой по продукту уже собран.

Следующий шаг для папки `newlook/` - не перепридумывание стратегии, а постепенная сборка операционной упаковки первого потока.

Ниже зафиксирован рабочий набор документов следующего слоя.

- `First-Flow-Launch-Blueprint.md` - главный документ запуска первого потока: цель, этапы `набор -> отбор -> интенсив -> переход в клуб -> первые 90 дней`, критерии успеха и точки проверки
- `First-Cohort-Selection-Playbook.md` - документ по набору и отбору первого потока: кого звать, как вести разговоры, по каким признакам брать или не брать человека
- `Participant-Journey.md` - карта пути участника от первого контакта до первых 90 дней в клубе: состояния, ожидания, риски выпадения и ключевые переходы
- `Intensive-Blueprint.md` - операционный дизайн входного интенсива: структура, логика блоков, ожидаемые сдвиги, признаки сильного первого опыта
- `Club-90-Day-Operating-Model.md` - модель первых 90 дней клуба: ритм, типы встреч, межсессионная механика, способы делать изменение видимым
- `Validation-and-Learning-System.md` - система проверки гипотез первого потока: какие сигналы и метрики смотреть, как собирать обратную связь и как принимать решения после запуска

## Рабочее различение
Каноника в этой папке отвечает на вопрос `что такое Новый взгляд`.

Следующий слой документов должен отвечать на вопрос `как запустить и доказать первый поток без потери канонической логики продукта`.
