## Назначение процесса

**Цель:** Ученик проходит путь от первого доступа до реферата, развивая разумное мышление.  
**Контур:** Академия (передача образца метода разумного мышления).  
**Владелец:** Администратор учебного процесса (операционная сторона).

---

## Участники (роли)

| Роль | Функция в процессе |
|------|--------------------|
| Администратор учебного процесса (АУчП) | Ведёт таблицу, инициирует организационные контакты, принимает решение о допуске |
| Куратор | Проверяет задания по регламенту, сообщает АУчП о прохождении урока |
| НМ | Преподаёт, ведёт вебинары; операционное сопровождение учеников — зона АУчП по регламенту |
| Ученик | Проходит уроки, сдаёт задания, участвует в вебинарах, пишет реферат |

---

## Схема процесса

```
[Оплата / Заявка подтверждена]  ← от П3 (привлечение)
            ↓
    ┌─────────────────────────────────────┐
    │           ОНБОРДИНГ                 │
    │  АУчП: выдаёт доступ,               │
    │  объясняет формат,                  │
    │  заключает учебный договор          │
    │  (ученик фиксирует личную цель)     │
    └─────────────────────────────────────┘
            ↓
    ┌─────────────────────────────────────┐
    │       САМОСТОЯТЕЛЬНОЕ ОБУЧЕНИЕ      │
    │  Ученик: читает, смотрит записи,    │
    │  посещает вебинары,                 │
    │  выполняет задания                  │
    │                                     │
    │  АУчП: ведёт таблицу непрерывно      │
    │  Триггер: нет активности 5 дней →   │
    │  АУчП пишет первым (не ждёт «пропал»)│
    └─────────────────────────────────────┘
            ↓
    ┌─────────────────────────────────────┐
    │       ПРОВЕРКА ЗАДАНИЯ              │  ← тариф «С куратором»
    │  Ученик: самооценка по чеклисту     │
    │  (до отправки куратору)             │
    │                                     │
    │  Куратор: проверяет строго          │
    │  по регламенту (≤ 2 итерации)       │
    │  → при 3-м возврате: эскалация АУчП  │
    │                                     │
    │  Куратор → АУчП: сообщает о          │
    │  прохождении урока                  │
    └─────────────────────────────────────┘
            ↓
    ┌─────────────────────────────────────┐
    │     ПРОМЕЖУТОЧНЫЙ РУБЕж (~50%)      │
    │  АУчП: инициирует чек-ин             │
    │  (ученик прошёл ≥ половины уроков)  │
    │  АУчП → ученику: «как движется?»     │
    │  Куратор → АУчП: общая картина сдач  │
    │  Артефакт: чек-ин протокол          │
    └─────────────────────────────────────┘
            ↓
    ┌─────────────────────────────────────┐
    │  КОНТРОЛЬ СРОКА (непрерывно)        │
    │  АУчП: за 3 дня и за 1 день          │
    │  до конца доступа →                 │
    │  уведомляет ученика и куратора      │
    └─────────────────────────────────────┘
            ↓
    [Все обязательные уроки сданы?]
    АУчП: принимает решение о допуске к реферату (по таблице)
            ↓
    ┌─────────────────────────────────────┐
    │     ПОДГОТОВКА К РЕФЕРАТУ           │
    │  Ученик: подаёт план реферата       │
    │  Куратор: одобряет план по          │
    │  регламенту (структура, тема)       │
    │  ← только структурная проверка,     │
    │     не содержательная оценка        │
    └─────────────────────────────────────┘
            ↓
    [Написание реферата]
            ↓
    [Защита / Завершение]
            ↓
    ┌─────────────────────────────────────┐
    │     РЕФЛЕКСИЯ НА ВЫХОДЕ             │
    │  Ученик: 10-15 мин структурированная│
    │  рефлексия (что изменилось в        │
    │  мышлении относительно цели         │
    │  из учебного договора)              │
    │  Артефакт: exit-запись              │
    └─────────────────────────────────────┘
            ↓
    [Переход на следующий уровень / Продление / Выход]
```

## Артефакты (work products)

| Артефакт | Кто ведёт | Что фиксирует |
|----------|-----------|---------------|
| Таблица Успеваемости | АУчП | ФИО, почта, ник ТГ, сданные уроки + даты, посещённые вебинары + даты |
| Учебный договор | АУчП (при онбординге) | Личная цель ученика, ожидаемый результат |
| Уведомление о сроке | АУчП → ученик + куратор | За 3 дня и за 1 день |
| Чек-ин протокол | АУчП | Состояние на ~50% пути: сдано N уроков, вебинары посещены |
| Самооценочный чеклист | Ученик | Отметка пунктов регламента до отправки куратору |
| Решение о допуске | АУчП | Да/Нет на основе таблицы |
| План реферата | Ученик → Куратор | Структура, тема, опорные тезисы |
| Реферат | Ученик | Итоговый документ |
| Exit-рефлексия | Ученик | Что изменилось в мышлении; связь с целью из учебного договора |

## Критические точки и правила

| Ситуация | Правило |
|----------|---------|
| Ученик не сдаёт задания | АУчП пишет первым — выясняет причину |
| Нет активности 5 дней | АУчП инициирует контакт проактивно (не ждёт «пропал») |
| Задание выполнено не полностью | Куратор возвращает — до выполнения всех обязательных пунктов, но не более 2 итераций |
| 3-я итерация возврата задания | Куратор сообщает АУчП — АУчП принимает организационное решение (не содержательное) |
| Срок доступа истекает | Уведомление за 3 дня и за 1 день; куратор сообщает АУчП о необходимости продления |
| Нужен ли допуск к реферату | АУчП решает по таблице — не по содержательной оценке |
| Операционная проблема ученика | АУчП решает самостоятельно — не эскалирует к носителю образца метода разумного мышления |
| Куратор проверяет план реферата | Только структурная проверка — не редактура мышления, не подсказка содержания |

## Мировые практики, лежащие в основе изменений

| Практика | Источник | Что добавлено |
|----------|----------|---------------|
| Учебный договор (Learning Contract) | Malcolm Knowles, андрагогика | Онбординг: ученик фиксирует личную цель |
| Mastery Learning (≤ 2 итерации) | Benjamin Bloom | Ограничение циклов проверки + эскалация |
| Early Warning System (5 дней) | Ed-tech / Learning Analytics | Проактивный контакт до исчезновения |
| Самооценка перед сдачей | Dylan Wiliam, Formative Assessment | Чеклист самооценки развивает метакогницию |
| Milestone checkpoint (~50%) | ADDIE / UbD (Wiggins & McTighe) | Промежуточный рубеж вместо линейного потока |
| Scaffolding перед рефератом | Vygotsky, Zone of Proximal Development | План реферата как промежуточная опора |
| Structured Exit Reflection | Schön, Reflective Practitioner | Рефлексия на выходе фиксирует обучение |

## Ключевые различения (из EP.SCHOOL-LEARNING-DOMAIN)

- **D.001** — Куратор ≠ Педагог
- **D.002** — Организационный контакт ≠ Педагогический контакт
- **D.003** — Допуск к реферату ≠ Оценка
- **D.005** — Онбординг ≠ Обучение

## Главный инвариант

> Операционные обращения ученика ведёт АУчП по регламенту П1.
