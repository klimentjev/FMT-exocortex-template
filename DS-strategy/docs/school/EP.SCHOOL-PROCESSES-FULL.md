## Обзор: четыре процесса и их связи

| # | Процесс | Контур | Цель | Владелец |
|---|---------|--------|------|----------|
| П1 | Учебный процесс (жизненный цикл ученика) | Академия | Ученик осваивает метод разумного мышления | АУчП |
| П2 | Контентный конвейер (вебинар → артефакт) | Лаборатория + Администрация | Знание сохраняется и становится доступным | Редактор + Методист |
| П3 | Привлечение участников | Администрация (внешний) | Новые ученики приходят в Академию | Менеджер по участникам |
| П4 | Разработка и тестирование продуктов | Лаборатория + Академия | Новые форматы выходят в свет | НМ + Тестировщик |

### Связи между процессами

```
П2 (Контент) ──────────────────→ П3 (Привлечение)
  ↑                          контент питает воронку
  │
  │ методические паттерны
  ↓
П4 (Продукты) ─────────────────→ П3 (Привлечение)
                              релиз → анонс и продажа
                              │
                              ↓
П3 (Привлечение) ──────────────→ П1 (Учебный цикл)
                              оплата → онбординг
                              │
                              ↓
П4 (Продукты) ─────────────────→ П1 (Учебный цикл)
                              если продукт встраивается в обучение
```

**Центральный узел — П2.** Без контентного конвейера: П3 не имеет материала для воронки, П4 не имеет методических паттернов, П1 не имеет шпаргалок для кураторов.

**Критически недоукомплектован — П3.** В команде нет Менеджера по работе с участниками и Менеджера по партнёрствам. Входящие вопросы уходят в содержательный контур (НМ) — прямое нарушение архитектуры, блокирует рост.

## П1 · Учебный процесс (жизненный цикл ученика)

**Цель:** Ученик проходит путь от первого доступа до реферата, развивая разумное мышление.
**Контур:** Академия.
**Владелец:** Администратор учебного процесса (АУчП).

### Задачи

- Обеспечить непрерывное движение каждого ученика по курсу
- Не допустить «пропадания» ученика без реакции системы
- Вести операционные обращения учеников через АУчП по регламенту, не смешивая операционный и содержательный контуры
- Довести каждого допущенного ученика до реферата

### Этапы

| # | Этап | Кто действует | Что происходит |
|---|------|---------------|----------------|
| 1 | **Онбординг** | АУчП | Выдаёт доступ, объясняет формат, заключает учебный договор (ученик фиксирует личную цель) |
| 2 | **Самостоятельное обучение** | Ученик + АУчП | Ученик читает, смотрит записи, посещает вебинары, выполняет задания. АУчП ведёт таблицу и пишет первым при отсутствии активности 5+ дней |
| 3 | **Проверка задания** | Куратор → АУчП | Ученик делает самооценку по чеклисту. Куратор проверяет строго по регламенту (≤2 итерации). При 3-м возврате — эскалация к АУчП. Куратор сообщает АУчП о прохождении |
| 4 | **Промежуточный рубеж (~50%)** | АУчП | Инициирует чек-ин: «как движется?». Куратор передаёт общую картину сдач |
| 5 | **Контроль сроков** | АУчП | Уведомляет ученика и куратора за 3 дня и за 1 день до конца доступа |
| 6 | **Допуск к реферату** | АУчП | Принимает решение по таблице успеваемости — не по содержательной оценке |
| 7 | **Подготовка к реферату** | Ученик → Куратор | Ученик подаёт план. Куратор одобряет только структуру — не редактирует мышление |
| 8 | **Написание и защита** | Ученик | Итоговый документ → защита |
| 9 | **Рефлексия на выходе** | Ученик | 10–15 мин: что изменилось в мышлении относительно цели из учебного договора |
| 10 | **Переход** | АУчП | Следующий уровень / продление / выход |

### Функции по ролям

| Роль | Функция | Чего НЕ делает |
|------|---------|----------------|
| **АУчП** | Ведёт таблицу успеваемости, пишет ученикам первым, контролирует сроки, принимает решение о допуске к реферату, самостоятельно решает типовые организационные вопросы | Не объясняет философию, не принимает содержательных решений, не эскалирует операционные вопросы к НМ без необходимости |
| **Куратор** | Проверяет задания по регламенту, сообщает АУчП о прохождении урока | Не пишет ученику первым, не помогает «думать», не объясняет Образец метода разумного мышления, не проводит педагогическую работу |
| **НМ** | Преподаёт, ведёт вебинары | Операционное сопровождение учеников — зона АУчП по регламенту П1 |

### Артефакты

| Артефакт | Кто ведёт | Что фиксирует |
|----------|-----------|---------------|
| Таблица успеваемости | АУчП | ФИО, почта, ник ТГ, сданные уроки + даты, посещённые вебинары + даты |
| Учебный договор | АУчП (при онбординге) | Личная цель ученика, ожидаемый результат |
| Уведомление о сроке | АУчП → ученик + куратор | За 3 дня и за 1 день |
| Чек-ин протокол | АУчП | Состояние на ~50% пути |
| Самооценочный чеклист | Ученик | Отметка пунктов регламента до отправки куратору |
| Решение о допуске | АУчП | Да/Нет на основе таблицы |
| План реферата | Ученик → Куратор | Структура, тема, опорные тезисы |
| Реферат | Ученик | Итоговый документ |
| Exit-рефлексия | Ученик | Что изменилось в мышлении; связь с целью из учебного договора |

### Критические точки

| Ситуация | Правило |
|----------|---------|
| Нет активности 5 дней | АУчП пишет первым — не ждёт «пропал» |
| Задание не выполнено полностью | Куратор возвращает — до выполнения всех обязательных пунктов, но не более 2 итераций |
| 3-я итерация возврата | Куратор сообщает АУчП — АУчП принимает организационное решение |
| Срок доступа истекает | Уведомление за 3 дня и за 1 день |
| Операционная проблема ученика | АУчП решает самостоятельно по регламенту — без выноса в содержательный контур без необходимости |

### Главный инвариант

> Операционные обращения ученика ведёт АУчП по регламенту П1.

### Ключевые различения

| Различение | Суть |
|-----------|------|
| Куратор ≠ Педагог | Куратор проверяет по регламенту — не ведёт феноменологическое развитие |
| Организационный контакт ≠ Педагогический контакт | АУчП пишет об организации — куратор отвечает на задания |
| Допуск к реферату ≠ Оценка | АУчП решает по таблице — не по качеству мышления |
| Онбординг ≠ Обучение | Онбординг — организационный вход, обучение начинается с первого урока |

## П2 · Контентный конвейер (вебинар → артефакт)

**Цель:** Каждый вебинар и лекция оставляет переиспользуемый след — текст, методика, база знаний.
**Контур:** Лаборатория (смысл) + Администрация (структурирование).
**Владелец:** Редактор (операционно), Методист (содержательно).

### Задачи

- Переводить знание из формата «в голове основателя» в воспроизводимые артефакты
- Формировать живую Базу знаний, пригодную для переиспользования в П1 и П3
- Создать «Учебник метода НИФ» — первичную формализацию методики
- Обеспечить П1 шпаргалками для кураторов, П3 — контентом для воронки

### Этапы

| # | Этап | Кто действует | Что происходит |
|---|------|---------------|----------------|
| 1 | **Источник** | НМ | Проводит вебинар или лекцию |
| 2 | **Запись и транскрипт** | Технический специалист | Запись сохраняется, транскрипт формируется |
| 3 | **Тайм-лайн аннотация** | Редактор | Временные коды + темы. Без анализа, без выводов — только разметка того, что звучало |
| 4 | **Структурирование** | Редактор | Читаемый текст из записи, тегирование по рубрикам, добавление в Базу знаний |
| 5 | **Методическая обработка** | Методист + НМ | Вычленяет ключевые понятия, ключевые переходы между понятиями, алгоритмы и схемы. Работает с аннотированным файлом от редактора, в паре с опытным участником |
| 6 | **Фиксация в Учебнике** | Методист | Паттерн или приём записывается в «Учебник метода НИФ» |
| 7 | **Опциональная публикация** | Редактор + Методист | Блог, рассылка, фрагмент книги → идёт в П3 |

### Функции по ролям

| Роль | Функция | Чего НЕ делает |
|------|---------|----------------|
| **Редактор** | Структурирует запись, создаёт тайм-лайн аннотацию, тегирует материалы, ведёт Базу знаний | Не интерпретирует смысл, не добавляет своё знание, не является соавтором Образца метода разумного мышления |
| **Методист** | Выявляет ключевые понятия и переходы, фиксирует алгоритмы и паттерны, пишет Учебник метода НИФ | Не передаёт метод ученикам, не меняет Образец метода разумного мышления единолично |
| **НМ** | Источник знания, работает в паре с методистом на этапе обработки | Не делает структурирование и аннотацию сам |
| **Технический специалист** | Обеспечивает запись и транскрипт | — |

### Артефакты

| Артефакт | Кто создаёт | Что содержит |
|----------|-------------|-------------|
| Тайм-лайн аннотация | Редактор | Временные коды + темы, без анализа |
| Структурированный текст | Редактор | Читаемый текст из записи вебинара |
| Запись в Базе знаний | Редактор | Тегированный материал по рубрикам |
| Паттерн / схема | Методист | Алгоритм или методический приём |
| Раздел «Учебника метода НИФ» | Методист | Формализованная методика |
| Публикация (блог/книга) | Редактор + Методист | Готовый к распространению текст |

### Критические точки

| Ситуация | Правило |
|----------|---------|
| Запись не обработана | Знание теряется — каждый вебинар должен пройти через редактора |
| Методическая находка только «в голове» НМ | Методист фиксирует — задача не откладывается |
| Текст обработан, но не тегирован | Редактор тегирует сразу — иначе Базу знаний невозможно использовать |
| Редактор интерпретирует смысл | Редактор структурирует, не интерпретирует — содержание исходит от НМ |

### Ключевые различения

| Различение | Суть |
|-----------|------|
| Редактор ≠ Соавтор | Структурирует чужое знание — не добавляет своё |
| Методист ≠ Преподаватель | Фиксирует метод — не передаёт его ученикам |
| База знаний ≠ Архив | База — живая, тегированная, переиспользуемая; архив — просто хранилище |
| Учебник метода ≠ Курс | Учебник описывает что существует; курс — как учить |

## П3 · Привлечение участников

**Цель:** Люди, которым нужно разумное мышление, узнают о школе и становятся учениками.
**Контур:** Администрация (внешний контур).
**Владелец:** Менеджер по работе с участниками.

### Задачи

- Обеспечить ответ на каждый входящий вопрос — ни один не теряется
- Сопроводить человека от интереса до решения о вступлении без давления
- Найти корпоративные контакты и передать их НМ для переговоров
- Поддерживать живую атмосферу в сообществе (удержание существующих)

### Этапы

**Поток B2C (частные ученики):**

| # | Этап | Кто действует | Что происходит |
|---|------|---------------|----------------|
| 1 | **Контент-источник** | Редактор + Методист (из П2) | Посты, рассылки, публикации формируют входящий поток |
| 2 | **Входящий вопрос** | — | Telegram / ВКонтакте / email / форма сайта |
| 3 | **Первый ответ** | Менеджер по участникам | Отвечает честно о форматах и тарифах. Не давит, не уговаривает |
| 4 | **Сопровождение** | Менеджер по участникам | Ведёт диалог с теми, кто не принял решение. Напоминает, помогает выбрать тариф |
| 5 | **Решение** | Человек → Менеджер | Да: передаёт заявку АУчП. Нет/позже: остаётся в контакте |
| 6 | **Оплата и онбординг** | АУчП | Оформляет доступ → запускает П1 |

**Поток B2B (корпоративные клиенты):**

| # | Этап | Кто действует | Что происходит |
|---|------|---------------|----------------|
| 1 | **Поиск контактов** | Менеджер по партнёрствам | Учебные заведения, корпорации, клубы, лектории |
| 2 | **Первый выход** | Менеджер по партнёрствам | Рассказывает о проекте, узнаёт интерес |
| 3 | **Тёплый контакт** | → НМ | Переговоры ведёт НМ |
| 4 | **Корпоративный контракт** | НМ | → Запускает П1 для группы |

### Функции по ролям

| Роль | Функция | Чего НЕ делает |
|------|---------|----------------|
| **Менеджер по участникам** | Отвечает на входящие вопросы, ведёт диалог до решения, помогает выбрать тариф, передаёт заявку АУчП | Не объясняет Образец метода разумного мышления, не давит, не уговаривает, не принимает оплату |
| **Менеджер по партнёрствам** | Ищет корпоративные контакты, делает первый выход | Не ведёт переговоры — только находит дверь |
| **Комьюнити-менеджер** | Оживляет сообщество, вовлекает участников в контакт друг с другом | Не отвечает на вопросы о тарифах и поступлении |
| **НМ** | Ведёт переговоры по B2B контрактам | Не занимается B2C-воронкой, не отвечает на типовые входящие вопросы |
| **АУчП** | Принимает заявку, оформляет оплату, запускает онбординг | — |

### Артефакты

| Артефакт | Кто создаёт | Что содержит |
|----------|-------------|-------------|
| Ответ на входящий вопрос | Менеджер | Честная информация о форматах и тарифах |
| Заявка на оплату | Менеджер → АУчП | ФИО, тариф, контакт |
| Контактная база B2B | Менеджер по партнёрствам | Список тёплых контактов с заметками |

### Критические точки

| Ситуация | Правило |
|----------|---------|
| Входящий вопрос остался без ответа | Менеджер отвечает — потерянный вопрос = потерянный ученик |
| Человек интересовался, но не решился | Менеджер ведёт диалог, напоминает — без давления |
| Попытка «продать» через убеждение | Запрещено — менеджер рассказывает честно, не уговаривает |
| Корпоративный интерес | Менеджер по партнёрствам находит дверь; переговоры ведёт НМ |
| Вопрос о содержании курса / методике | Менеджер не интерпретирует Образец метода разумного мышления — передаёт факты о форматах |

### Ключевые различения

| Различение | Суть |
|-----------|------|
| Информирование ≠ Продажа | Рассказывает честно — не «закрывает сделку» любым путём |
| Входящий интерес ≠ Готовность платить | Интерес ≠ решение; менеджер сопровождает до решения |
| B2C ≠ B2B воронка | B2C: менеджер ведёт до оплаты. B2B: менеджер находит дверь, НМ ведёт переговоры |
| Ответ на вопрос ≠ Объяснение Образца метода разумного мышления | Менеджер отвечает на организационные вопросы — не объясняет суть метода |

## П4 · Разработка и тестирование продуктов

**Цель:** Новые форматы (игры, курсы, методики) доводятся до рабочего состояния и выходят в свет.
**Контур:** Лаборатория (идея + методика) + Академия (тестирование).
**Владелец:** НМ (содержание), Тестировщик (механика).

### Задачи

- Довести АнтиСофист до состояния «берёшь и играешь без объяснений» — дедлайн апрель 2026
- Создать Методичку куратора как переиспользуемый продукт
- Обеспечить передачу методических паттернов из П2 в продуктовую логику
- Не допустить упрощения Образца метода разумного мышления ради «удобства» продукта

### Этапы

| # | Этап | Кто действует | Что происходит |
|---|------|---------------|----------------|
| 1 | **Идея** | НМ | Источник идеи и содержания — Образец метода разумного мышления не адаптируется под рынок |
| 2 | **Определение формата** | НМ + Методист | Настолка / цифровой прототип / курс / методичка |
| 3 | **Первый прототип** | НМ + Методист | Содержание, правила, структура |
| 4 | **Итерации тестирования** | Тестировщик | Играет / проходит на разных конфигурациях участников. Фиксирует: где ломается, что непонятно. Предлагает правки |
| 5 | **Финализация** | Все участники | Правила и контент доведены до критерия готовности |
| 6 | **Релиз** | → П3 + П1 | Передаётся в привлечение для анонса и продажи; при необходимости встраивается в учебный процесс |

### Функции по ролям

| Роль | Функция | Чего НЕ делает |
|------|---------|----------------|
| **НМ** | Источник идеи и содержания, участвует в прототипировании, принимает финальные решения по содержанию | Не тестирует механику сам, не адаптирует Образец метода разумного мышления под «удобство» |
| **Методист** | Формализует методику, участвует в разработке прототипа, привносит паттерны из П2 | Не принимает решения по Образцу метода разумного мышления единолично |
| **Тестировщик** | Тестирует механику на разных конфигурациях, фиксирует сбои, предлагает правки | Не решает, правильно ли Образец метода разумного мышления передан — это вопрос содержания, не механики |

### Артефакты

| Артефакт | Кто создаёт | Что содержит |
|----------|-------------|-------------|
| Прототип продукта | НМ + Методист | Первая версия правил / контента |
| Отчёт о тестировании | Тестировщик | Зафиксированные сбои, непонятные места, предложения по правкам |
| Финальная версия | Все | Готовый к распространению продукт |

### Критические точки

| Ситуация | Правило |
|----------|---------|
| Тестировщик не находит сбоев | Возможно, тест нерепрезентативен — проверить разнообразие конфигураций |
| Сбой механики найден | Фиксировать немедленно, не замалчивать из вежливости |
| Продукт «почти готов» до бесконечности | Установить критерий готовности заранее — дедлайн или «N сессий без критических сбоев» |
| Дедлайн провален | Принять решение: перенос релиза или закрытие направления (АнтиСофист — апрель 2026) |
| Содержание меняется в ходе тестирования | Изменения Образца метода разумного мышления — только через НМ, не по итогам «удобства» |

### Ключевые различения

| Различение | Суть |
|-----------|------|
| Механика ≠ Содержание | Тестировщик проверяет механику — не решает, правильно ли Образец метода разумного мышления передан |
| Прототип ≠ Релиз | Прототип — для итераций; релиз — готов к использованию без объяснений |
| Тест ≠ Демо | Тест: найти что не работает. Демо: показать что работает |
| Адаптация под удобство ≠ Развитие продукта | Упрощение Образца метода разумного мышления ради «удобства» — запрещено (EP.SCHOOL-ENV §IV) |
