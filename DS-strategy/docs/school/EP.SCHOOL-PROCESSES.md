## П1. Учебный процесс (жизненный цикл ученика)

**Цель:** Ученик проходит путь от первого контакта к промежуточным отчетным работам и к диплому, развивая разумное мышление.

**Контур:** Академия (передача Образца метода разумного мышления).

```
Заявка / оплата
      ↓
Онбординг (доступ, знакомство с форматом)
      ↓
Самостоятельное прохождение уроков
      ↓ (тариф «Чтение и обсуждение»), может быть привлечен куратор
Проверка заданий Куратором по регламенту
      ↓
Контроль прогресса (Администратор учебного процесса ведёт таблицу)
      ↓
Уведомление о сроках доступа (за 3 дня и за 1 день)
      ↓
Допуск к реферату (решает Администратор учебного процесса по таблице)
      ↓
Написание и защита промежуточной отчетной работы (например: реферата)
      ↓
Переход на следующий уровень / продление
```

**Критические точки:**
- Ученик «пропадает» на неделю → Администратор учебного процесса пишет первым
- Задание не выполнено → Куратор работает до выполнения всех обязательных пунктов
- Срок доступа истекает → уведомление боту + решение о продлении

**Чего не должно быть:** операционные обращения ученика стабильно выносятся мимо АУчП в содержательный контур без необходимости.

## П2. Контентный конвейер (вебинар → артефакт)

**Цель:** Из каждого вебинара / лекции создаются переиспользуемые артефакты — текст, методика, база знаний.

**Контур:** Лаборатория (смысл) + Администрация (структурирование).

```
Вебинар / лекция проведён (НМ)
      ↓
Запись + транскрипт
      ↓
Редактор: структурирует в читаемый текст
Редактор: тегирует по рубрикам, добавляет в Базу знаний
      ↓
Методист: выявляет алгоритмы, схемы, паттерны
Методист: фиксирует в «Учебнике метода НИФ» (в разработке)
      ↓
Опционально: подготовка к публикации (блог, книга)
```

**Критические точки:**
- Запись без обработки = знание теряется
- Методические находки только «в голове» основателя → риск невоспроизводимости

**Результат процесса:** Каждый вебинар оставляет след в Базе знаний и, со временем, в методике.

## П3. Привлечение участников

**Цель:** Люди, которым нужно разумное мышление, узнают о школе и становятся учениками.

**Контур:** Администрация (внешний контур).

```
Контент (посты, рассылки, мероприятия, сарафанное радио)
      ↓
Входящий вопрос (Telegram, ВКонтакте, email)
      ↓
Менеджер по работе с участниками: отвечает, объясняет форматы, помогает выбрать тариф
      ↓
Решение о вступлении → передача заявки на оплату Администратору учебного процесса
      ↓
Оплата → онбординг (переход в П1)
      ↓
(параллельно) Менеджер по партнёрствам: корпоративные и институциональные контакты
      → тёплый контакт передаётся НМ для переговоров
```

**Критические точки:**
- Входящий вопрос без ответа = потерянный ученик
- «Давление» при продаже → противоречит миссии; Менеджер рассказывает честно, не уговаривает

## П4. Разработка и тестирование продуктов

**Цель:** Новые форматы (игры, курсы, методики) доводятся до рабочего состояния и выходят в свет.

**Контур:** Лаборатория (идея) + Академия (тестирование).

```
Идея продукта (НМ)
      ↓
Прототип (совместно с Методистом / Тестировщиком)
      ↓
Итерации тестирования (Тестировщик фиксирует сбои механики)
      ↓
Финализация правил / контента
      ↓
Релиз (передаётся в П3 для привлечения)
```

**Текущий экземпляр:** Игра АнтиСофист → дедлайн апрель 2026.
