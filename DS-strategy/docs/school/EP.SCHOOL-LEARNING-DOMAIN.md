## Stage 01 · Domain Selection

```
Domain:       Управление учебным процессом («ФИЛОСОФ&Я»)
Scope:        Жизненный цикл ученика — от онбординга до завершения/реферата
Exclusions:   содержание Образца метода разумного мышления, привлечение новых учеников, разработка продуктов
Practitioners: Администратор учебного процесса, Куратор, Ученик
Adjacent:     Передача Образца метода разумного мышления (Академия/Лаборатория), Продажи/Привлечение, Разработка методики
```

**Проверка (SPF 01): это домен, не тема**

| Элемент домена | Наличие |
|---------------|---------|
| Practitioners | ✅ АУчП, Куратор, Ученик |
| Methods | ✅ проверка по регламенту, допуск к реферату, уведомление о сроках |
| Work products | ✅ таблица Успеваемости, допуск к реферату, реферат |
| Failure modes | ✅ ученик пропадает, куратор подменяет педагога, операционные обращения выносятся в содержательный контур (НМ) вместо АУчП |

### D.001 · Куратор vs. Педагог

**Определение:**  
Куратор — роль, функция которой оценивать выполнение заданий строго по заданным критериям (регламенту). Педагог — роль, функция которой передавать метод разумного мышления.

**Тест различения:** Кто принял решение «достаточно ли выполнено задание» — по критериям (Куратор) или по пониманию мышления ученика (Педагог)?

| Куратор | vs. | Педагог |
|---------|-----|---------|
| Проверяет по регламенту | | Передаёт Образец метода разумного мышления |
| Не объясняет философию | | Объясняет, помогает понять |
| Не пишет ученику первым | | Ведёт ученика содержательно |
| Решает: «пункты выполнены?» | | Решает: «мышление развивается?» |

**Типичная путаница:** Куратор начинает «помогать думать» — объяснять смысл заданий, подсказывать ответы.

**Почему важно:** Подмена куратором педагога нарушает методику передачи Образца метода разумного мышления — ученик получает чужую интерпретацию вместо собственного движения.

**Implied FM:** Куратор-педагог (куратор выходит за границы своей функции).

### D.002 · Организационный контакт vs. Педагогический контакт

**Определение:**  
Организационный контакт — инициатива АУчП: статус ученика, сроки, напоминания, проблемы доступа. Педагогический контакт — инициатива Куратора (или Носителя Образца метода разумного мышления): обратная связь по содержанию задания.

**Тест различения:** Причина контакта — «что у тебя со сдачей» (орг.) или «вот что ты сделал/не сделал по регламенту» (пед.)?

| Организационный | vs. | Педагогический |
|----------------|-----|---------------|
| Инициирует АУчП | | Инициирует Куратор |
| Тема: сроки, доступ, статус | | Тема: задание, регламент |
| Пишет первым | | Отвечает на сданную работу |

**Типичная путаница:** Куратор тормошит ученика — «напоминает» о заданиях; АУчП комментирует работы.

**Почему важно:** Если Куратор берёт организационные контакты — АУчП не нужен, процесс перегружает Куратора, ученики получают двойные сигналы.

**Implied FM:** Инверсия контактов (куратор тормошит, АУчП молчит).

### D.003 · Допуск к реферату vs. Оценка

**Определение:**  
Допуск к реферату — административное решение АУчП на основе объективных данных таблицы (уроки сданы / не сданы). Оценка — педагогическое суждение о качестве мышления ученика.

**Тест различения:** Принимающий решение смотрит в таблицу (допуск) или в содержание работ (оценка)?

| Допуск к реферату | vs. | Оценка |
|------------------|-----|--------|
| Принимает АУчП | | Принимает Куратор / Педагог |
| Критерий: факт сдачи пунктов | | Критерий: качество мышления |
| Алгоритмизируем | | Требует суждения |

**Типичная путаница:** АУчП начинает судить «готов ли ученик содержательно» — выходит за рамки своей функции.

**Почему важно:** Размытие ведёт к субъективным и непоследовательным решениям о допуске, к эскалации к Носителю Образца метода разумного мышления.

**Implied FM:** АУчП как педагог (административная роль принимает педагогические решения).

### D.004 · Учебный процесс vs. Образовательный процесс

**Определение:**  
Учебный процесс — операционная сторона: кто, когда, что сдал, в каком статусе. Образовательный процесс — содержательная сторона: как развивается мышление ученика.

**Тест различения:** Можно ли зафиксировать это в таблице с датами и фактами (учебный) — или требует феноменологического наблюдения (образовательный)?

| Учебный процесс | vs. | Образовательный процесс |
|----------------|-----|------------------------|
| Управляет АУчП | | Управляет Педагог/Куратор |
| Артефакт: таблица Успеваемости | | Артефакт: запись развития, рефлексия |
| Объективируем | | Требует суждения |

**Типичная путаница:** «Учебный процесс настроен» = «ученики развиваются». Это ложно: налаженная операционка не гарантирует образования.

**Почему важно:** Опасность принять организационную исправность за педагогический успех.

**Implied FM:** Подмена результата (Operations ≠ Learning Outcome).

---

### D.005 · Онбординг vs. Обучение

**Определение:**  
Онбординг — технический ввод ученика в среду: доступ, ориентация, первые организационные шаги. Обучение — содержательное движение ученика через Образец метода разумного мышления.

**Тест различения:** Задача — «ученик знает, куда идти и как устроена среда» (онбординг) или «ученик движется в мышлении» (обучение)?

| Онбординг | vs. | Обучение |
|-----------|-----|---------|
| Технический | | Содержательный |
| Ограничен во времени (разовый) | | Непрерывен весь период |
| Ответственность: АУчП | | Ответственность: Куратор + Педагог |
| Артефакт: доступ выдан, формат объяснён | | Артефакт: задание сдано, урок пройден |

**Типичная путаница:** На онбординге начинают «учить» — объяснять суть метода. Ученик перегружается до начала обучения.

**Почему важно:** Смешение перегружает ученика и деформирует первое впечатление от Образца метода разумного мышления.

**Implied FM:** Педагогизация онбординга.
