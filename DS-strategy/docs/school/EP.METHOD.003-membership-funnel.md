# Воронка для переживаемых членских продуктов

## 1. Определение
Четырёхуровневая воронка для продуктов, ценность которых раскрывается через участие (клубы, сообщества, живые курсы). Каждый уровень самостоятелен по ценности и органично ведёт к следующему.

## 2. Проблема
Прямая продажа высокочекового членства тёплой аудитории даёт низкую конверсию: люди знают автора, но не знают формат изнутри.

## 3. IPO-паттерн

| Элемент | Описание |
|---------|----------|
| Входы | Тёплая аудитория (подписчики, читатели); высокочековый переживаемый продукт |
| Обработка | Последовательное снижение барьера входа через 4 уровня |
| Выходы | Конверсия в платное членство; пассивный доход от контентного продукта |

## 4. Уровни воронки

| Уровень | Формат | Ценность для клиента | Цель уровня |
|---------|--------|----------------------|-------------|
| 0. Бесплатный контент | Канал/блог | «Здесь думают иначе» | Охват, доверие |
| 1. Пассивный продукт | PDF/запись/гайд | Глубина без обязательств | Доход без участия; создаёт FOMO по живому |
| 2. Живой пробник | Разовая сессия/воркшоп | Опыт формата изнутри | Закрыть барьер «не знаю, что там» |
| 3. Членство | Регулярный формат | Принадлежность + регулярность | Основной доход |

## 5. Ключевые принципы
- Каждый уровень продаёт **следующий**, а не просто даёт контент
- Пробник — не демо, а **полноценная сессия**: человек должен чувствовать, что получил ценность даже без конверсии
- Оффер членства — в конце пробника, не в начале: сначала опыт, потом предложение
- Антихёрн в членстве: ценность сообщества растёт со временем → уход = потеря накопленного контекста

## 6. Мосты между уровнями

| Переход | Механизм |
|---------|----------|
| Контент → Пассивный продукт | Пост: «всё это собрано целиком в [продукте]» |
| Пассивный → Пробник | Последняя страница продукта: приглашение на живую сессию |
| Пробник → Членство | Последние 5-10 мин сессии: честный рассказ о клубе, без давления |

## 7. Предмасштабный gate

Для переживаемого трансформационного продукта рост нельзя масштабировать раньше, чем доказана причинная цепочка:

`точный набор -> сильный первый опыт -> желание остаться -> удержание -> рекомендации`

Если цепочка не доказана, масштабирование не решает проблему, а ускоряет слабую модель.

### Что проверять до масштабирования

- долю сильных откликов на личный призыв;
- конверсию из разговора в участие;
- качество обратной связи после первого опыта;
- конверсию из первого опыта в членство / продолжение;
- удержание 3 месяца;
- удержание 6 месяцев;
- количество сильных личных рекомендаций;
- качество людей, пришедших по рекомендации.

### Следствие

На раннем этапе важнее всего оптимизировать не охват, а качество этой цепочки. Трафик, маркетинг и масштабная воронка включаются только после того, как продукт доказал, что:
1. приводит правильных людей;
2. даёт им реальный сдвиг;
3. удерживает их;
4. порождает рекомендации.
