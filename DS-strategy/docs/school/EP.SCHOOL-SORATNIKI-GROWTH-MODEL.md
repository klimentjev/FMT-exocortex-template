> **Ключевая идея:** Соратники — не помощники и не подрядчики. Каждый соратник — **деятель** (MIM.D.012): создаёт шедевр (MIM.WP.002), который меняет реальность школы, при наставничестве НМ. Работа в школе — это одновременно вклад в школу и собственное развитие как мыслителей и созидателей.

---

## I. Переосмысление роли соратника через МИМ

### Три уровня участника ФиЯ × три программы МИМ

| Уровень (EP.SCHOOL-ENV) | Аналог в МИМ | Программа развития | Главный формат |
|------------------------|--------------|-------------------|----------------|
| Ученик (потребитель) | Проходит практикум | Личное развитие (MIM.PRG.001) | Практикум (MIM.FMT.002) |
| Адепт (практик) | Деятель | Рабочее развитие (MIM.PRG.002) | Резидентура (MIM.FMT.003) |
| Носитель метода | Мастер (MIM.R.004) | Исследовательское (MIM.PRG.003) | Семинар (MIM.FMT.001) |

**Соратник работает в режиме деятеля.** Он не просто помогает школе — он создаёт шедевр (рабочий результат) при наставничестве НМ. Это меняет всё: мотивацию, формат встреч, критерии оценки.

### Критическое различение

**MIM.D.009 Мастерская ≠ Школа:**
ФиЯ — среда бесконечного развития, а не курс с дипломом. Соратники не «проходят стажировку» — они входят в мастерскую, где нет выпуска, есть следующий уровень.

**MIM.D.012 Мыслитель ≠ Созидатель:**
Соратник, который участвует в обсуждениях, читает документы, думает о ролях — но не создаёт артефактов, изменяющих реальность школы — не является созидателем. Шедевр — это то, что меняет реальность школы, а не внутреннее состояние соратника.

## II. Шедевры четырёх кругов (MIM.WP.002)

> **Шедевр** — внешне предъявляемый рабочий результат с критериями, заданными до начала работы. Не учебный проект — настоящий рабочий артефакт, который предъявляется внешнему миру и используется дальше.

### Круг 1 · Кураторский → П1

**Шедевр:** Методичка куратора — полный регламент работы с учеником по всем урокам блока «Античность» с шпаргалками по каждой теме.

**Критерии готовности (заданы заранее):**
- Куратор может проверить задание по любому уроку Античности без консультации с НМ
- В методичке есть: критерии проверки каждого пункта задания, типичные ошибки учеников, ключевые вопросы для возврата задания
- Новый куратор, прочитав методичку, может начать работу без дополнительного обучения

**Наставник:** НМ — только для содержательных вопросов об образце метода разумного мышления
**Операционный наставник:** Операционный директор — для организационных вопросов

---

### Круг 2 · Редакторский → П2

**Шедевр:** Аннотированная база вебинаров по блоку «Античность» + первый раздел живой Базы знаний.

**Критерии готовности:**
- Каждый вебинар блока «Античность» имеет тайм-лайн аннотацию: временные коды + темы
- База знаний тегирована — можно найти любую тему за 2 минуты
- Любой новый соратник открывает базу и находит ключевые темы вебинара без прочтения транскрипта

**Первый шедевр (пилот):** Полная аннотация вебинаров по теме «Милецы»
**Критерии пилота:** 5+ вебинаров аннотированы, материал по одному вебинару сверен методистом и признан пригодным

**Наставник по содержанию:** НМ (в паре с методистами на стыке П2)

---

### Круг 3 · Методический → П2 + П4

**Шедевр:** Первый раздел Учебника метода НИФ — «Античность: ключевые понятия, переходы и шпаргалки для кураторов».

**Критерии готовности:**
- По каждой теме Античности зафиксированы: ключевые понятия (которые ученик должен усвоить), ключевые переходы (которые нельзя пропустить), 3–5 вопросов-шпаргалок для куратора
- Куратор, прочитав шпаргалку по теме, может направить ученика к ключевому переходу без объяснения всей лекции НМ
- Документ используется в реальной работе кураторов (не «лежит»)

**Первый шедевр (пилот):** Полная методическая обработка темы «Милецы» — ключевые понятия + переходы + шпаргалки
**Наставник:** НМ (работает в плотном контакте)

---

### Круг 4 · Привлечения → П3

**Шедевр:** Работающая B2C-воронка без участия НМ в первой линии — скрипты ответов на типовые вопросы + первые 10 лидов, доведённых до решения.

**Критерии готовности:**
- НМ не участвовал ни в одном B2C-контакте в течение 2 недель подряд
- Есть скрипты ответов на 10 самых частых вопросов о форматах и тарифах
- Каждый входящий вопрос получает ответ в течение 4 часов

**Второй шедевр (B2B):** Контактная база из 5 тёплых корпоративных контактов с первым выходом
**Наставник:** НМ — только для корпоративных переговоров, не для B2C-воронки

## III. Операционный ритм кругов: инженерный цикл (MIM.M.003)

> «У нас не обучение с оценками, у нас развитие с разборами.»

Каждый круг работает по инженерному циклу развития. Цикл повторяется каждые **2 недели**.

```
ИНЖЕНЕРНЫЙ ЦИКЛ РАЗВИТИЯ (MIM.M.003)

Обновление портфеля проблем
      ↓  «Что нового сломалось? Что мешает двигаться?»
Проблематизация
      ↓  «Что именно не работает? Почему?»
Стратегирование
      ↓  «Как решим? Какой формат, какой срок?»
Проверка
      ↓  (2 недели работы)
Замер
      ↓  «Что получилось? Что замерили?»
(→ обновление портфеля проблем следующего цикла)
```

### Применение по кругам

| Круг | Типичный портфель проблем | Что замеряется |
|------|--------------------------|----------------|
| Кураторский | Ученик застрял на задании, непонятны критерии проверки, нет шпаргалки по теме X | Количество аннотированных тем, % заданий проверенных без вопросов к НМ |
| Редакторский | Нет доступа к архиву, непонятен формат аннотации, слишком большой файл | Количество аннотированных вебинаров, время на один вебинар |
| Методический | Не понял что «ключевой переход», методика не применяется куратором | Количество обработанных тем, используются ли шпаргалки кураторами |
| Привлечения | Вопрос завис без ответа, не знаю как ответить на X, потенциальный корпоративный клиент | Время ответа на входящий, конверсия интерес→оплата |

## IV. Система разборов (MIM.FMT.004 + MIM.M.002)

> **MIM.D.004 Разбор ≠ Экзамен:**
> Разбор — замер + обратная связь непрерывно. Экзамен — оценка в конце. В мастерской нет режима «сейчас учимся, потом проверяем». Каждый разбор — одновременно развитие и замер.

### Три уровня разборов

**Уровень 1: Внутри круга (каждые 2 недели, 45 мин)**

| Этап | Время | Что происходит |
|------|-------|----------------|
| Показ работы | 10 мин | Один участник показывает конкретный артефакт (аннотацию, шпаргалку, ответ клиенту) |
| Разбор наставником (или старшим) | 15 мин | Что работает, что нет, где ошибка — с объяснением почему |
| Замер квалификации (MIM.WP.003) | 5 мин | Где человек сейчас по трём шкалам (агентность, масштаб, дисциплина) |
| Следующий шаг | 5 мин | Конкретное действие до следующего разбора |

**Важно:** Приносить сырой материал, не «причёсанный». Разбор работает только с реальным, а не подготовленным для показа. Если приносишь идеальное — разбор теряет смысл.

**Уровень 2: Межкруговой (раз в месяц, 60 мин)**

Участники: НМ + старшие кругов (выявляются в процессе, см. раздел V).

| Этап | Время | Что происходит |
|------|-------|----------------|
| Статус шедевров | 20 мин | Каждый круг: где шедевр, что блокирует |
| Разбор одного шедевра | 25 мин | Один круг показывает промежуточный результат, НМ разбирает по содержанию |
| Обновление критериев | 15 мин | Нужно ли скорректировать критерии шедевра на основе практики |

**Уровень 3: Наставничество НМ ↔ Методический круг (еженедельно, 30 мин)**

Методический круг — ближайший к НМ. Только здесь разбор происходит еженедельно, потому что содержание требует постоянной сверки.

## V. Квалификационная рамка: выявление лидеров кругов (MIM.M.004)

> Лидер круга не назначается. Он выявляется через работу по трём шкалам. Первостепенна агентность.

### Три шкалы

| Шкала | Вопрос | Что наблюдаем |
|-------|--------|---------------|
| **Агентность** ⭐ | Инициирует сам или ждёт задания? | Видит проблему без подсказки → предлагает решение → делает. Не анализирует, а действует (MIM.D.011) |
| **Масштаб** | Каков радиус влияния? | Своя задача → своя роль → весь круг → вся школа |
| **Методологическая дисциплина** | Следует и улучшает регламенты? | Работает по установленному регламенту + предлагает улучшения |

### Как замеряем

На каждом разборе (уровень 1) старший фиксирует по трём шкалам:

```
Агентность:
  □ Пришёл с тем, что было задано (0)
  □ Сделал задание + заметил проблему рядом (1)
  □ Решил проблему, о которой его не просили (2)
  □ Инициировал улучшение процесса для всего круга (3)

Масштаб:
  □ Работает только в рамках своей задачи (0)
  □ Помогает другим участникам круга (1)
  □ Влияет на процесс всего круга (2)
  □ Видит связь своей работы с целями школы в целом (3)

Методологическая дисциплина:
  □ Не знает или игнорирует регламент (0)
  □ Следует регламенту (1)
  □ Следует и улучшает регламент через предложения (2)
  □ Документирует улучшения для других (3)
```

**Лидер круга = тот, кто первым достигает агентности уровня 2–3 + масштаба уровня 2.**

Не самый опытный. Не самый умный. Тот, кто инициирует и чьи инициативы влияют на весь круг.

## VI. Интеллектуальные партнёры (MIM.M.005)

> Применяется в Редакторском и Методическом кругах — там, где качество интеллектуальной работы критично и нельзя проверить «автоматически».

### Трёхуровневый цикл мышления

**Уровень 1: Индивидуальная рефлексия (перед встречей, 15 мин)**
Каждый участник письменно отвечает:
- Что я сделал? (конкретный артефакт)
- Что меня в нём беспокоит? (где сомневаюсь)
- Какой вопрос хочу обсудить?

Письменность обязательна — мышление письмом ≠ мышление в голове (MIM.D.012).

**Уровень 2: Парная рефлексия (на встрече, 15 мин)**
Участники обмениваются артефактами в парах:
- Читаю артефакт партнёра
- Задаю один вопрос: «Где я не понял, что имеется в виду?»
- Даю обратную связь: «Вот что я увидел как ключевое. Совпадает с твоим намерением?»

**Уровень 3: Коллективная рефлексия (на встрече, 15 мин)**
Пары делятся паттернами:
- Что повторялось как ошибка у нескольких?
- Что оказалось неожиданно хорошим?
- Что меняем в регламенте?

### Применение в Редакторском круге

Перед первым разбором по Милецам:
1. Каждый редактор присылает свою аннотацию за день до встречи
2. На встрече: обмен аннотациями в парах → парная рефлексия
3. Коллективно выявляем стандарт: что в аннотации «хорошо» и что «нет»
4. Этот стандарт становится регламентом для дальнейших аннотаций

### Применение в Методическом круге

Именно так создаётся Учебник метода НИФ:
1. Методист делает черновик «ключевых переходов» по теме
2. Второй участник (Варвара или НМ) читает и задаёт вопросы: «Что означает этот переход? Как куратор узнает, что ученик его прошёл?»
3. Коллективно — итоговая версия шпаргалки

## VII. НМ как наставник (MIM.R.001)

> Переход от роли «Соло-оператора» к роли Наставника — самое важное изменение в личной операционной системе.

### Что делает Наставник

| Делает | Не делает |
|--------|-----------|
| Ведёт разборы (ведёт, а не присутствует) | Выдаёт задания кругам |
| Даёт обратную связь по содержанию | Контролирует исполнение задач |
| Замеряет квалификацию на каждом разборе | Отслеживает статусы ежедневно |
| Задаёт вопросы, которые развивают | Отвечает на вопросы, которые можно найти в регламенте |
| Определяет критерии шедевров | Проверяет артефакты редакторов (это задача Главного редактора) |

### Расписание наставничества НМ

| Формат | Кто | Ритм | Время |
|--------|-----|------|-------|
| Разбор с Методическим кругом | Лариса, Вероника (+ Варвара) | Еженедельно | 30 мин |
| Межкруговой разбор | Старшие всех кругов | Раз в месяц | 60 мин |
| Разбор шедевра по запросу | Любой круг | По потребности | 30 мин |

**Разбор Методического круга — приоритет.** Это ближайший контакт с образцом метода разумного мышления. Именно здесь создаётся Учебник метода НИФ. Пропускать нельзя.

## VIII. Режим ошибок, которого нужно избегать (MIM.FM.002)

> **MIM.FM.002 Заучивание ошибок:** Отсутствие разборов приводит к накоплению и закреплению незамеченных ошибок. Чем дольше нет разбора — тем глубже заученные ошибки.

### Конкретные риски для каждого круга

| Круг | Типичная незамеченная ошибка | Последствие без разбора |
|------|------------------------------|------------------------|
| Кураторский | Куратор даёт «педагогические» комментарии вместо регламентных | Ученики получают объяснения вместо направления к самостоятельному мышлению |
| Редакторский | Редактор интерпретирует смысл вместо разметки | База знаний содержит субъективные суждения, а не факты |
| Методический | Методист фиксирует «красивые формулировки» вместо рабочих шпаргалок | Куратор не может использовать шпаргалку в работе |
| Привлечения | Менеджер начинает «убеждать» вместо «информировать» | Противоречие миссии школы в первом контакте |

### Противоядие: разбор должен начаться ДО того, как ошибки закрепились

**Правило:** Первый разбор каждого круга — **не позже чем через 7 дней после начала работы**.

Нельзя ждать «когда будет что показать». Разбор работает именно с сырым, неготовым — это и есть его смысл.

## IX. Интеграция в первый вебинар команды

Это изменяет структуру первого вебинара. Добавляются три обязательных элемента:

### Что добавить к предыдущему сценарию

**1. В начале — назвать шедевр каждого круга (5 мин)**
Не задачи, не роли — именно шедевры с критериями. Люди должны знать, что они создадут.

**2. В блоке «Первое общее дело» — разобрать различение разбор ≠ экзамен (5 мин)**
> «Первый разбор каждого круга — через 7 дней. Приносите сырой материал. Нас интересует не "красиво" — нас интересует реальное. Разбор — это не экзамен. Это развитие + замер одновременно.»

**3. В закрытии — каждый называет свой первый шаг К ШЕДЕВРУ (не просто "что сделаю")**
Не «я напишу аннотацию» — а «я сделаю аннотацию первого вебинара по Милецам как первый шаг к нашему шедевру — полной базе Античности».

Контекст шедевра меняет мотивацию: человек делает не задание — он создаёт то, что останется.

## XI. Критические различения для команды

Эти различения нужно назвать явно на первом вебинаре — иначе они будут нарушаться молча.

| Различение | Почему критично для ФиЯ |
|-----------|------------------------|
| **MIM.D.004 Разбор ≠ Экзамен** | Если воспринять разбор как экзамен — будут приносить «причёсанное». Разбор умрёт |
| **MIM.D.011 Агентность ≠ Аналитическая установка** | Соратник, который думает, обсуждает, читает, но не создаёт артефактов — не является созидателем школы |
| **MIM.D.012 Мыслитель ≠ Созидатель** | «Написал план» ≠ «Сделал». Шедевр — это то, что предъявляется внешнему миру |
| **MIM.D.009 Мастерская ≠ Школа** | Соратники входят в мастерскую бесконечного развития, а не в программу с «окончанием» |

## XII. Личная траектория роста в каждой роли

> Работа в школе — не стажировка и не волонтёрство. Каждая роль создаёт особый тип мыслителя и деятеля. Это не написано в описании задач — это проявляется через работу.

Общая логика: Адепт входит в роль → работает над шедевром → разбор обнаруживает что он не понимает → он углубляет понимание → постепенно становится носителем этого уровня знания. Путь к Носителю метода идёт через конкретную роль, а не мимо неё.

---

### Куратор → «Тот, кто понимает метод изнутри»

**Что реально развивается:**
Точность обратной связи — умение задать вопрос, который возвращает ученика к самостоятельному мышлению, вместо того чтобы объяснять.

**Ключевое открытие в роли:**
Правило «не объяснять философию» — не ограничение, а испытание. Оно вскрывает: понимаешь ли ты материал достаточно глубоко, чтобы не объяснять? Большинство начинающих кураторов хотят помочь объяснением — потому что глубинного понимания ещё нет. Когда понимание появляется — объяснение перестаёт быть нужным.

**Траектория:**

| Этап | Признак |
|------|---------|
| Вход | Прошёл значительную часть курса, понимает задания |
| Рост | Учится задавать вопросы вместо ответов; углубляет знание материала через работу с чужими ошибками |
| Зрелость | Видит, где именно ученик потерял нить — и может вернуть его туда одним вопросом |
| Потолок роли | Может обучать новых кураторов; понимает метод на уровне «могу объяснить почему задание устроено именно так» |

**Путь к Носителю метода:** Куратор, доработавший до зрелости, — это человек, который понимает метод изнутри через работу с сотнями заданий. Это ближайший путь к роли Методиста.

---

### Редактор → «Тот, кто видит архитектуру знания»

**Что реально развивается:**
Структурное мышление — способность найти скелет в сложном тексте: что здесь главное, что развёртка, что пример.

**Ключевое открытие в роли:**
По мере аннотирования вебинаров начинаешь видеть паттерны в том, как НМ думает и движется от темы к теме. Это знание, которое невозможно получить, просто слушая вебинары — оно появляется только когда разбираешь их «по косточкам» десятки раз.

**Траектория:**

| Этап | Признак |
|------|---------|
| Вход | Умеет работать с текстами, любит систематизировать |
| Рост | Учится разделять «философское содержание» и «структуру подачи»; видит как одна и та же идея появляется в разных вебинарах |
| Зрелость | Становится хранителем интеллектуальной памяти школы — знает, где и когда НМ говорил о конкретной теме лучше, чем сам НМ |
| Потолок роли | Может сам предложить структуру нового раздела базы знаний; видит пробелы в материале |

**Путь к Носителю метода:** Редактор, увидевший архитектуру знания, — следующий кандидат в Методисты. Он накопил фактуру; Методист научится её осмыслять как метод.

---

### Методист → «Тот, кто может объяснить как работает метод»

**Что реально развивается:**
Понимание метода НИФ на уровне реконструкции: не «я знаю что делать», а «я понимаю почему это работает и могу объяснить другому».

**Ключевое открытие в роли:**
При попытке написать шпаргалку для куратора — обнаруживаешь что сам не понимаешь перехода, который пытаешься описать. Работа методиста постоянно вскрывает иллюзию понимания. Именно поэтому разбор с НМ здесь еженедельный — не как контроль, а как необходимость.

**Траектория:**

| Этап | Признак |
|------|---------|
| Вход | Прошёл курс, хочет понять метод глубже, а не просто его применять |
| Рост | Учится формулировать «ключевые переходы» — что именно ученик должен увидеть в теме |
| Зрелость | Пишет шпаргалки, которые куратор реально использует; может объяснить почему задание устроено именно так |
| Потолок роли | Соавтор Учебника метода НИФ — может описать метод как систему для человека, незнакомого с курсом |

**Путь к Носителю метода:** Методист — прямой путь. Единственная роль, где работа буквально состоит в том, чтобы стать носителем метода на операциональном уровне.

---

### Менеджер по участникам → «Тот, кто передаёт дух школы в первом контакте»

**Что реально развивается:**
Коммуникативная точность — умение понять что нужно человеку и честно сказать, подходит ли ему школа, без давления и без искажения.

**Ключевое открытие в роли:**
Нельзя «продавать» школу так же, как продают другие курсы. Человек чувствует неаутентичность. Единственный работающий инструмент — сам веришь, сам понимаешь ценность, говоришь честно. Это превращает роль в школу аутентичной коммуникации.

**Траектория:**

| Этап | Признак |
|------|---------|
| Вход | Любит общаться, сам верит в проект, понимает его ценность |
| Рост | Учится слушать что человек ищет — и соотносить это с тем, что школа реально даёт |
| Зрелость | Может за 5 минут диалога понять — подойдёт ли школа этому человеку; не «продаёт», а «помогает решить» |
| Потолок роли | Может самостоятельно вести B2B-переговоры первого уровня; видит паттерны в том, кто приходит и почему |

**Путь к Носителю метода:** Косвенный, но реальный. Менеджер, который прошёл 100 разговоров о школе, — глубоко понимает, что в ней ценного и для кого. Это понимание ценнее, чем у тех, кто просто прошёл курс.

---

### Администратор учебного процесса → «Тот, кто видит школу как живой организм»

**Что реально развивается:**
Системное мышление через операционную практику — видеть учебный процесс целиком: кто застрял, где узкое место, что нужно починить до того, как оно сломалось.

**Ключевое открытие в роли:**
Администратор видит паттерны в успеваемости учеников, которые не видит никто — в том числе НМ. Оказывается, большинство уходят не потому что «не понравилось», а потому что не получили напоминания в нужный момент. Это знание превращает администратора в стратега удержания.

**Траектория:**

| Этап | Признак |
|------|---------|
| Вход | Любит порядок, умеет держать несколько процессов |
| Рост | Видит где процесс даёт сбой — не по жалобам, а по данным |
| Зрелость | Предлагает улучшения системы до того, как она даёт сбой |
| Потолок роли | Готов к роли Операционного директора — управляет не задачами, а процессами |

**Путь к Носителю метода:** Не прямой, но важный. Администратор, понявший как учебный процесс работает изнутри, — незаменим при масштабировании. Понимание «почему процесс устроен именно так» приходит через работу с живыми учениками.

---

### Операционный директор → «Координатор операционного контура»

> Полные требования: **EP.SCHOOL-OPERATIONAL-DIRECTOR.**

**Что реально развивается:**
Управление процессами (а не задачами): видение стыков П1–П4, узких мест, регламентов; способность не эскалировать к НМ то, что решается операционно.

**Ключевое различение:**
Операционный директор не владелец одного процесса — он координатор всех четырёх. АУчП владеет П1; операционный директор следит за согласованностью П1–П4 и за тем, чтобы операционный контур закрывал типовые кейсы по регламенту.

**Траектория:**

| Этап | Признак |
|------|---------|
| Вход | Вырос из АУчП: видит учебный процесс целиком, готов отвечать за ритм и регламенты |
| Рост | Ведёт операционные синки с владельцами процессов, закрывает блокеры без эскалации к НМ |
| Зрелость | Операционный наставник Кураторского круга; регламенты актуальны, стыки процессов без сбоев |
| Потолок роли | Операционные вопросы закрываются в контуре Администрации по регламенту; к НМ — только внерегламентное |

**Путь к Носителю метода:** Не прямой. Операционный директор — системная роль Администрации; содержание и метод остаются зоной НМ и методического контура.

```
                    НОСИТЕЛЬ МЕТОДА
                         ↑
          ┌──────────────┴──────────────┐
          │                             │
       Методист                    (будущее)
          ↑                             
    ┌─────┴──────┐                      
    │            │                      
 Куратор     Редактор                   
 (изнутри)   (архитектура)              
    
    Менеджер → понимает ценность школы → Носитель духа
    
    Администратор → видит организм целиком → Операционный директор
```

**Принцип:** Роль — это не просто задачи для школы. Это среда, в которой определённый тип мышления развивается через реальную работу. Шедевр — внешний результат. Тип мышления — личный результат.
