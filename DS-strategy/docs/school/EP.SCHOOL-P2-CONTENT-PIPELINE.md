## Назначение процесса

**Цель:** Каждый вебинар и лекция оставляет переиспользуемый след — текст, методика, база знаний.  
**Контур:** Лаборатория (смысл) + Администрация (структурирование).  
**Владелец:** Редактор (операционно), Методист (содержательно).

---

## Участники (роли)

| Роль | Функция в процессе |
|------|--------------------|
| НМ | Проводит вебинар — источник знания |
| Редактор | Структурирует запись в текст, тегирует, ведёт Базу знаний |
| Методист | Выявляет алгоритмы и паттерны, пишет «Учебник метода НИФ» |

---

## Схема процесса

```
[Вебинар / Лекция проведён (НМ)]
            ↓
    [Запись + транскрипт]
            ↓
    ┌──────────────────────────────┐
    │          Редактор            │
    │  — структурирует в текст     │
    │  — тегирует по рубрикам      │
    │  — добавляет в Базу знаний   │
    └──────────────────────────────┘
            ↓
    ┌──────────────────────────────┐
    │          Методист            │
    │  — выявляет алгоритмы        │
    │  — фиксирует схемы, паттерны │
    │  — пишет «Учебник метода НИФ»│
    └──────────────────────────────┘
            ↓
    [Опционально: публикация]
    — блог, книга, курс
            ↓
    [Артефакт доступен → переиспользуется в П1 и П3]
```

---

## Артефакты (work products)

| Артефакт | Кто создаёт | Что содержит |
|----------|-------------|-------------|
| Текст вебинара | Редактор | Структурированный читаемый текст |
| Запись в Базе знаний | Редактор | Тегированный материал по рубрикам |
| Паттерн / схема | Методист | Алгоритм или методический приём |
| Раздел «Учебника метода НИФ» | Методист | Формализованная методика |
| Публикация (блог/книга) | Редактор + Методист | Готовый к распространению текст |

---

## Критические точки и правила

| Ситуация | Правило |
|----------|---------|
| Запись не обработана | Знание теряется — каждый вебинар должен пройти через Редактора |
| Методическая находка только «в голове» Носителя Образца метода разумного мышления | Методист фиксирует — задача не откладывается |
| Текст обработан, но не тегирован | Редактор тегирует сразу — иначе Базу знаний невозможно использовать |
| Редактор интерпретирует смысл | Редактор структурирует, не интерпретирует — содержание исходит от Носителя Образца метода разумного мышления |

---

## Ключевые различения

| Различение | Суть |
|-----------|------|
| Редактор vs. Соавтор | Редактор структурирует чужое знание — не добавляет своё |
| Методист vs. Преподаватель | Методист фиксирует метод — не передаёт его ученикам |
| База знаний vs. Архив | База — живая, тегированная, переиспользуемая; архив — просто хранилище |
| Учебник метода vs. Курс | Учебник — описание метода (что существует); курс — downstream (как учить) |

---

## Почему процесс критичен

> За 13 лет работы школа накопила уникальный методический материал, который существует только в записях вебинаров и в голове основателя.  
> Без этого процесса знание невоспроизводимо при смене поколений.

---

## Связь с другими процессами

- → **П1**: готовые тексты и методика используются в учебном процессе
- → **П3**: обработанный контент идёт в блог и рассылки для привлечения участников
- → **П4**: методические паттерны питают разработку новых продуктов
