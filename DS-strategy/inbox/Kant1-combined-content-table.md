# Kant 1: объединенная таблица (рилсы + тексты)

Все позиции из двух файлов:
- `Kant1-content-shortlist.md` (первая транскрипция: `Kant 1 U.txt`)
- `Kant1V-content-shortlist.md` (вторая транскрипция: `Kant 1 V.txt`)

**Статус (колонка «Статус»):** `идея` → `в работе` (монтаж / черновик текста) → `опубликовано`. При необходимости: `отложено`, `отменено`.

**Ссылка:** полный URL ролика (YouTube Shorts, Reels, TG и т.д.) или URL поста в блоге; пока пусто — `—`.

### Импорт в Google Таблицы

**Полный материал (рекомендуется):** **[Kant1-combined-google-sheets-full.tsv](Kant1-combined-google-sheets-full.tsv)** — те же 9 колонок, в **«Полный материал»** лежит весь текст для рилса (формат, крючок, титры, описание, CTA, хэштеги + единый мост) или для поста (H1, лид, план, CTA + мост). Многострочные ячейки в кавычках — Google при импорте обычно кладёт их в одну ячейку.

**Краткий (только заголовки и таймкоды):** [Kant1-combined-google-sheets.tsv](Kant1-combined-google-sheets.tsv).

**Пересборка полного файла:** `node Kant1-gen-google-full.mjs` из папки `inbox/` (нужен Node.js).

Импорт полного TSV:

1. Google Таблицы → **Файл → Импортировать → Загрузить** → `Kant1-combined-google-sheets-full.tsv`.
2. Разделитель: **табуляция**; кодировка **UTF-8** (файл с BOM — Excel/Sheets нормально читают).
3. Если колонки «поехали»: при импорте включить опцию учёта кавычек (text in quotes) — в Google это обычно по умолчанию для `.tsv`.

Либо открыть `.tsv` в редакторе, выделить всё, вставить в **A1** — строки с многострочными полями иногда ломаются; надёжнее именно **Импорт файла**.
