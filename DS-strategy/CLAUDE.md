# DS-strategy — Инструкции для ассистента

> **Тип репозитория:** `DS/governance` (personal)

## Назначение

Личный стратегический хаб. Здесь живут:
- Неудовлетворённости и приоритеты
- Агрегированные планы из всех репозиториев
- Сессии стратегирования
- Ревью (дневные, недельные, месячные)

## Роль: Стратег (R1)

При работе в этом репо ассистент исполняет роль **Стратег** (R1).

### Ключевые задачи

1. **Стратегирование** — превращение неудовлетворённостей в приоритеты и РП
2. **Планирование** — агрегация `WORKPLAN.md` из всех репо в единый план
3. **Ревью** — сбор изменений и результатов по неделе

### WORKPLAN.md — Hub-and-Spoke

Каждый репозиторий в `c:\Users\admin\IWE\` содержит `WORKPLAN.md` в корне с текущими РП.

**Агрегация:** при создании плана дня/недели Стратег:
1. Обходит `c:\Users\admin\IWE\*\WORKPLAN.md`
2. Собирает РП со статусом pending/in-progress
3. Формирует агрегированный план в `current/`

**Обратная синхронизация:** при обновлении приоритетов Стратег:
1. Обновляет `current/WeekPlan W{N} YYYY-MM-DD.md` в этом репо
2. Обновляет соответствующие `WORKPLAN.md` в целевых репо

## Правила

1. **Не хранить предметные знания** — это Pack
2. **Учёт ≠ планирование** — факты отдельно от намерений
3. **Inbox при закрытии сессии:** при работе с файлами из `inbox/` — в Закрытии сессии спросить: использованные в сессии файлы оставить или удалить?
4. **Англоязычные термины в markdown:** писать целиком латиницей (`Ubiquitous Language`, `pull request`), без подмены букв визуально похожими символами кириллицы (ошибка вида «Уbiquitous»).

## Структура файлов

| Директория | Содержание |
|------------|------------|
| `docs/` | Долгоживущие документы: Strategy.md, Dissatisfactions.md, Session Agenda.md |
| `current/` | Регулярно обновляемые: WeekPlan, WeekReport, DayPlan |
| `archive/` | Старые планы и отчёты |
| `inbox/` | Временное сырьё: скрипты обработки, выгрузки, reference-заметки из чатов — приходит извне, обрабатывается |
| `library/` | Постоянные файлы источников (тексты первоисточников): PDF и TXT книг по авторам (`hegel/`, `platon/`, `kant/`, `linkov/`, `muravev/`, `deutsch/`) |
| `drafts/` | Авторские документы в работе: планы, черновики статей, отчёты — до перехода в `docs/` |
| `exocortex/` | Backup memory/ + CLAUDE.md |

**Различения:**
- **library** — файлы первоисточников (сами тексты). **PACK-bibliography** — знание о первоисточниках (карточки SRC.*, цитаты, аннотации); путь в workspace: `PACK/PACK-bibliography/`.
- Терминология: для библиотеки — «тексты первоисточников» или «файлы источников»; для Pack — «первоисточник доменного знания».

## Понедельник: пост в блог Маргариты Тяжевой

Регулярная задача из `docs/Strategy.md` (1 ч по понедельникам).

### При открытии дня (Day Open), если сегодня **понедельник**

В конце ответа по открытию дня ассистент **выводит готовую фразу-шаблон** — пользователь копирует её в чат (с подстановкой номера и темы), когда садится писать пост:

```
Пишем пост №___ для блога Маргариты Тяжевой, тема: ___.
Подтяни контекстом и опирайся на три файла-источника:
DS-strategy/drafts/INDEX-blog-margarita-tyazheva.md
DS-strategy/inbox/Блог Маргариты Тяжевой.docx
DS-strategy/inbox/Еже ИФ текст лекций (2025).pdf (локально; индекс `library/klimentiev/INDEX-ezhe-if-lectures-2025.md`, карточка `library/klimentiev/SRC.ezhe-if-text-lectures-2025-pdf.md`)
Нужны полная версия в drafts/ и сокращённая под TG (отдельный *-TG.md).
Опционально для углубления по теме: выборки из DS-strategy/library/muravev/lectures-an-full-text.txt.
```

### При выполнении задачи «написать пост»

1. **Обязательно** использовать контекст **всех трёх** файлов выше (`.docx` — извлечь текст из `word/document.xml` внутри архива docx или эквивалент).
2. Обновить `drafts/INDEX-blog-margarita-tyazheva.md` после появления нового поста.
3. Формат артефактов: полный черновик `.md` + версия под Telegram (как `blog-tyazheva-post-08-zenon.md` и `blog-tyazheva-post-08-zenon-TG.md`).
